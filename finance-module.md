# Finance Module â€” Summary & Frontend Guide

## What the module does

The Finance module implements the MIZAN costing system for MIZAN Architecture firm.
It calculates how much each project actually costs the company in labour, overhead, and equipment â€” and compares those costs against the contract value (Plan vs Fact).

### Core business logic
```
Hourly rate per employee =
  (Salary + Bonus + Tax 12% + JSSM 12% + Admin share + Equipment share + Overhead share)
  Ă· Working hours per month (176)

Admin share = Total ADM overhead Ă· Number of production employees
Equipment share = Total monthly amortization Ă· Number of production employees
Overhead share = Other overhead Ă· Number of production employees

Project MIZAN cost (Plan) = Total estimated hours Ă— Avg team hourly rate Ă— Risk coefficient (1.15)
Project Fact cost = Sum of monthly actual time-entry costs
```

### Two scheduled jobs (no UI needed)
| Schedule | What it does |
|---|---|
| Every night at 02:00 | Recalculates hourly rates for all employees using that month's overhead, equipment amortization, and exchange rate |
| 1st of every month at 03:00 | Rolls up the previous month's time entries into `user_monthly_allocation` and `project_monthly_costs` |

## Frontend UI recommendations

### Settings / Reference Data pages (admin only)

These pages let admins maintain the lookup tables used by the cron jobs.

1. **Exchange rates** â€” table with date + rate + source; date picker for new entries; inline edit.
2. **Overhead costs** â€” monthly grid grouped by category; one row per category per month; bulk entry for a whole month is helpful.
3. **Equipment** â€” equipment list with amortization/month shown per row; form to add/decommission; total amortization summary card at the top.

### Employee Hourly Rates page (admin / manager)

- Show the **current effective rate** for each employee (from `GET /finance/hourly-rates`).
- Expand row to see history.
- Rows with `source` = automatic (created by cron) â†’ **read-only badge**, no edit button.
- Admin can click **"Set manual rate"** â†’ opens form to create a new row with `POST /finance/hourly-rates`.

### Project Finance tab (inside each project)

Add a "Finance" tab to the project detail page with three sections:

1. **Contract & Plan** â€” shows current plan card (contract value, MIZAN cost, margin %). Button: "Recalculate plan" â†’ `POST /finance/projects/:id/plan`.
2. **Monthly cost history** â€” read-only table from `GET /finance/projects/:id/monthly-costs`. Columns: Month, Hours, Cost (UZS), Cost (USD), Employees, Finalized.
3. **Plan vs Fact chart** â€” line or bar chart comparing plan burn curve vs actual cumulative cost. Data from `GET /finance/projects/:id/plan-vs-fact`.

### Finance dashboard (manager / admin)

A summary page using the analytics endpoints:

- **Overview cards** â€” total payroll, overhead, equipment, gross margin for current month (`/analytics/finance-overview`).
- **Project profitability table** â€” sortable by margin or cost variance (`/analytics/project-profitability`).
- **Department cost breakdown** â€” bar chart by department (`/analytics/department-cost`).
- **Employee cost breakdown** â€” sortable table with hours + cost per person (`/analytics/employee-cost-breakdown`).

### Read-only indicators

The following data is machine-generated. The UI should show it as read-only (no edit/delete actions):

- Rows in **Hourly rates** that were created by the nightly cron (you can identify them because they will have no `notes` and `effectiveDate` = the date the cron ran, or simply treat all cron-created rows as non-editable by hiding edit controls for non-admin users).
- All rows in **Monthly costs** (`project_monthly_costs`, `user_monthly_allocation`).
- All rows in **Time entry costs** (`time_entry_costs`).
